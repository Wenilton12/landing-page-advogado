# MNADV - Advocacia Patrimonial | Holding Familiar

Este é o repositório da Landing Page profissional desenvolvida para o escritório **MNADV**, especializado em Holding Familiar, Planejamento Sucessório e Societário.

A página foi construída com foco em alta conversão, responsividade e design premium (estilo Dark Mode com detalhes em dourado), garantindo uma excelente experiência de usuário (UX) em dispositivos móveis e desktops.

## 🚀 Tecnologias Utilizadas

- **HTML5:** Semântico e estruturado.
- **Tailwind CSS (CDN):** Estilização ágil, utilitária e responsiva. Configuração customizada no próprio HTML para paleta de cores da marca.
- **JavaScript (Vanilla):** Lógica para os carrosséis de imagens e depoimentos, além do menu responsivo (mobile).
- **CSS Customizado (`style.css`):** Para ajustes finos (como ocultar barras de rolagem).

## 📌 Funcionalidades

- **Menu Responsivo:** Menu hamburger para dispositivos móveis com efeito de backdrop blur.
- **Carrosséis Interativos:**
  - Carrossel da estrutura do escritório com auto-play e navegação por setas.
  - Carrossel de depoimentos de clientes com pausa ao passar o mouse (hover) ou toque na tela.
- **Design Premium:** Efeitos de hover, glassmorphism, animações flutuantes e sombras suaves.
- **Integração Visual:** Embed de mapa do Google Maps e direcionamento estratégico para o WhatsApp (Call to Action).

## 📁 Estrutura de Arquivos

```text
/
├── assets/           # Imagens otimizadas (WebP) e SVGs do projeto
├── css/              # Arquivo style.css com customizações de scrollbar e utilidades
├── js/               # Arquivo main.js com a lógica de carrossel e menu mobile
├── index.html        # Estrutura principal da Landing Page
└── README.md         # Documentação do projeto
```

## 🛠️ Como Executar o Projeto

Como é um projeto puramente frontend estático, não há necessidade de build ou instalação de dependências.

1. Faça o clone ou o download deste repositório.
2. Abra o arquivo `index.html` em qualquer navegador moderno.
3. Para uma experiência de desenvolvimento melhor (live reload), recomenda-se usar a extensão **Live Server** no VS Code.
